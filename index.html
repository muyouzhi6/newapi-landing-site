🚀 Codex 接入教程

按系统找到配置目录，复制两段配置即可。不会暴露任何内部信息。

配置目录

  • Windows: C:\Users\你的用户名\.codex
  • Linux / macOS: ~/.codex
建议先备份原文件，再粘贴下方内容。

推荐模型

默认示例：gpt-5.2 + xhigh（可后续按需改）
wire_api = responses requires_openai_auth = true
config.toml
写入到 ~/.codex/config.toml
model = "gpt-5.2"
model_reasoning_effort = "xhigh"
model_provider = "newapi"

[model_providers.newapi]
name = "newapi"
base_url = "https://newapi.165201.xyz/v1"
wire_api = "responses"
requires_openai_auth = true
auth.json
写入到 ~/.codex/auth.json
{
  "OPENAI_API_KEY": "这里填入你刚创建的 Key"
}
常见问题
401：Key 错误/前后空格
JSON 报错：auth.json 不是合法 JSON
连接失败：确认可访问 https://newapi.165201.xyz/v1

❓ FAQ

把买家最常遇到的问题提前写清楚，减少来回沟通。

我应该怎么开始试用？
创建 Key → 按上方教程配置 Codex → 直接开始调用即可。
能不能限制 Key 的模型范围？
可以，在令牌管理里给 Key 设置模型限制与权限策略。
我需要对接 OpenAI SDK / 其他客户端怎么办？
直接使用同一 base_url：https://newapi.165201.xyz/v1，鉴权用你的 Key。
